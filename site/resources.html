FAQ

Is SGAIP secure?

SGAIP uses proven cryptographic standards (Ed25519 + SHA-256). The protocol has been carefully designed and documented. See the threat model for security analysis.

Can I use SGAIP in production?

Yes! The reference implementations are production-ready with comprehensive test suites. However, carefully review the threat model and security considerations for your use case.

Is SGAIP patent-free?

Yes. SGAIP uses only standard, unpatented cryptography (Ed25519 and SHA-256). The protocol is open and free to implement and use.

What if I lose my private key?

You cannot recover a lost private key. Generate a new key pair and transition to the new identity. This is a security feature‚Äîif a key could be recovered, it wouldn't be secure.

Can I change my identity?

Your identity is mathematically derived from your public key and cannot be changed. However, you can generate a new key pair and create a new identity at any time.

Does SGAIP require an internet connection?

No. Identity derivation and signature verification are completely offline. You don't need internet to create an identity or verify a signature.
